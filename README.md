## Design engineer building brands and web experiences for high-growth startups.

I arrived at the intersection of design and engineering because I couldn’t settle for doing only one. A product is only as good as its usability. Designing without execution leads to friction. Building without user empathy leads to products no one needs.

I work at the intersection of both to create digital experiences that work for everyone. Every decision serves a clear purpose.

> *“Complexity is a symptom. Clarity is the fix.”*

-----

## Approach

Design and engineering run concurrently — not in sequence.

|Phase                            |What it means                                                                                                                                |
|---------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------|
|**Audit & Discovery**            |Finding the real problem, not just the stated brief. I evaluate existing systems to map constraints before any solution work begins.         |
|**Prototyping as Design**        |Static screens hide edge cases. I build functional prototypes to validate interaction, state, and logic before committing to production code.|
|**Brand & Interface Engineering**|Every visual decision is backed by a technical rationale. Accessibility and measured performance are baseline requirements — not add-ons.    |
|**Delivery & Measurement**       |The work ships with measured results, structured documentation, and explicitly named tradeoffs.                                              |

**On AI:** I use it to scaffold component logic, format raw research data, and write baseline test coverage. It compresses the time between a decision and its implementation. It does not design the system.
